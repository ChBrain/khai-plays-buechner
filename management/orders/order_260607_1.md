# Management Order: Decouple Management Order Gating from Local Tests

## Direction

Staging plays in the playhouses must be an act of creative freedom, independent of top-down system infrastructure gating. Currently, play-staging is mixed with governance gating via local checks and staging orders. We must delegate all plan/order checklist verification to the engine and remove staging-specific orders from the playhouses.

## Orders

1. The Theatre Manager (franzos) shall remove all staging-specific management orders from the repository.
2. The Theatre Manager (franzos) shall remove the custom completeness test from the playhouse test suite.
3. The Theatre Manager (franzos) shall update `@chbrain/khai-tests` to the latest version to enable top-down validation of any future system-level plans.

## Implementation

- Delete all files under `management/orders/` except this order.
- Remove the `every management order is completed` test from `tests/house.test.mjs`.
- Run `npm test` to verify conformance.
